# Подробное объяснение конфигурации nginx

## Общая структура конфига

```nginx
server {
    # Директивы сервера
}
```

**`server`** - блок, который определяет виртуальный сервер. Nginx может обслуживать несколько сайтов одновременно, каждый описывается своим блоком `server`.

## Построчное объяснение

### 1. Настройка порта и протокола

```nginx
listen 8001 ssl;
```

**`listen`** - указывает на каком порту и по какому протоколу nginx будет принимать соединения.
- **`8001`** - номер порта. Выбран вместо стандартного 443, чтобы не требовать root права.
- **`ssl`** - включает SSL/TLS шифрование для этого порта. Без этого параметра nginx работал бы только по HTTP.

### 2. Имя сервера

```nginx
server_name localhost;
```

**`server_name`** - определяет, для каких доменных имен будет отвечать этот виртуальный сервер.
- **`localhost`** - nginx будет обрабатывать запросы, где в заголовке Host указано "localhost".
- Если бы было несколько доменов: `server_name example.com www.example.com;`

### 3. Путь к SSL сертификату

```nginx
ssl_certificate /etc/nginx/ssl/selfsigned.crt;
```

**`ssl_certificate`** - указывает путь к файлу SSL сертификата (публичная часть).
- Сертификат содержит публичный ключ и информацию о владельце.
- Отправляется клиенту для проверки подлинности сервера.
- Формат: обычно .crt, .pem или .cer файлы.

### 4. Путь к приватному ключу

```nginx
ssl_certificate_key /etc/nginx/ssl/selfsigned.key;
```

**`ssl_certificate_key`** - указывает путь к файлу с приватным ключом SSL сертификата.
- Приватный ключ используется для расшифровки данных, зашифрованных публичным ключом.
- Должен быть доступен только серверу (права доступа 600).
- НИКОГДА не передается клиенту.

### 5. Поддерживаемые SSL протоколы

```nginx
ssl_protocols TLSv1.2 TLSv1.3;
```

**`ssl_protocols`** - указывает какие версии протоколов SSL/TLS поддерживает сервер.
- **`TLSv1.2`** - стабильная и безопасная версия, поддерживается всеми современными браузерами.
- **`TLSv1.3`** - новейшая версия с улучшенной безопасностью и производительностью.
- Старые версии (SSLv3, TLSv1.0, TLSv1.1) исключены из-за уязвимостей.

### 6. Алгоритмы шифрования

```nginx
ssl_ciphers HIGH:!aNULL:!MD5;
```

**`ssl_ciphers`** - определяет какие алгоритмы шифрования можно использовать.
- **`HIGH`** - использовать только сильные алгоритмы шифрования (128+ бит).
- **`!aNULL`** - исключить алгоритмы без аутентификации (небезопасные).
- **`!MD5`** - исключить алгоритмы с хэшированием MD5 (устаревшие).
- Формат: список через двоеточие, где `!` означает исключение.

### 7. Блок обработки запросов

```nginx
location / {
    # Директивы обработки
}
```

**`location /`** - определяет как обрабатывать запросы к определенным URL.
- **`/`** - обрабатывать все запросы (корневой путь и все подпути).
- Можно создать несколько location блоков: `location /api/`, `location /static/` и т.д.

### 8. Проксирование запросов

```nginx
proxy_pass http://127.0.0.1:8000;
```

**`proxy_pass`** - перенаправляет входящие запросы на другой сервер.
- **`http://127.0.0.1:8000`** - адрес и порт Express сервера.
- **`127.0.0.1`** - localhost (локальный адрес).
- **`8000`** - порт на котором работает Node.js приложение.
- nginx выступает как reverse proxy между клиентом и Express.

### 9. Передача заголовка Host

```nginx
proxy_set_header Host $host;
```

**`proxy_set_header Host $host`** - передает оригинальный заголовок Host от клиента к Express серверу.
- **`$host`** - переменная nginx с именем хоста из запроса клиента.
- Без этого Express получал бы Host: 127.0.0.1:8000 вместо Host: localhost:8001.
- Важно для приложений, которые зависят от имени хоста.

### 10. Передача реального IP клиента

```nginx
proxy_set_header X-Real-IP $remote_addr;
```

**`proxy_set_header X-Real-IP $remote_addr`** - передает реальный IP адрес клиента.
- **`$remote_addr`** - переменная nginx с IP адресом клиента.
- **`X-Real-IP`** - нестандартный заголовок для передачи оригинального IP.
- Без этого Express видел бы только IP nginx (127.0.0.1).

### 11. Цепочка прокси серверов

```nginx
proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
```

**`proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for`** - сохраняет цепочку всех прокси серверов.
- **`X-Forwarded-For`** - стандартный заголовок для отслеживания маршрута запроса.
- **`$proxy_add_x_forwarded_for`** - добавляет текущий IP к существующему списку.
- Формат: `203.0.113.1, 198.51.100.1, 192.0.2.1` (через запятую).

### 12. Информация о протоколе

```nginx
proxy_set_header X-Forwarded-Proto $scheme;
```

**`proxy_set_header X-Forwarded-Proto $scheme`** - сообщает Express серверу какой протокол использовал клиент.
- **`$scheme`** - переменная nginx: "http" или "https".
- **`X-Forwarded-Proto`** - стандартный заголовок для передачи протокола.
- Важно для приложений, которые должны знать использовался ли HTTPS.

## Принцип работы

1. **Клиент** подключается к `https://localhost:8001`
2. **nginx** принимает HTTPS соединение, расшифровывает трафик
3. **nginx** проксирует запрос к Express на `http://127.0.0.1:8000`
4. **Express** обрабатывает запрос и отправляет ответ nginx
5. **nginx** шифрует ответ и отправляет клиенту

## Преимущества такой архитектуры

- **Безопасность**: SSL терминация на nginx
- **Производительность**: nginx эффективно обрабатывает статические файлы и SSL
- **Масштабируемость**: можно легко добавить балансировку нагрузки
- **Гибкость**: разные приложения на разных портах под одним доменом
